# AI 智能监控能力训练与打包协作流程

## 目标

把客户现场常见的 NVR 品牌、录像回查方式和专项 AI 能力沉淀为可复用能力包，最终上传到 AgentHub 能力市场。

## 角色分工

你负责提供现场业务样本和验证反馈：

- 目标场景：例如 AI 找人、物品遗失、录像回查、高空抛物。
- 设备信息：NVR 品牌、型号、固件版本、摄像头通道、内网访问方式。
- 样本材料：脱敏截图、录像片段、典型问题描述、期望回答格式。
- 验证反馈：哪些结果正确、哪些误报、哪些现场表达需要补充。

我负责整理训练资产并打包平台能力：

- 归类能力类型和适配器类型。
- 整理训练样本、提示词、工具参数和测试用例。
- 设计能力配置模板和本地执行动作契约。
- 收集代码、配置、说明文档和版本信息。
- 打包上传到 AgentHub，并协助线上验证。

## 每个能力包需要收集的材料

### 1. 基础信息

- 能力名称：例如“大华 NVR 录像回查适配器”。
- 所属模块：AI 智能监控。
- 能力类型：录像回查、AI 找人、物品遗失、车辆检索等。
- 支持设备：品牌、型号、固件版本。
- 使用限制：是否需要固定 IP、是否需要管理员账号、是否支持跨通道查询。

### 2. 接口与配置

- NVR 地址、端口、协议类型。
- 登录方式和权限要求。
- 录像查询接口。
- 录像片段下载或回放取流方式。
- 通道号与摄像头位置映射方式。

### 3. 训练样本

- 用户自然语言问题。
- 结构化解析结果。
- 应调用的工具动作。
- 工具返回样例。
- 最终回答样例。

示例：

```json
{
  "user": "查一下大厅下午3点左右谁拿走了桌上的手机",
  "intent": "lost_item",
  "camera_location": "大厅",
  "time_range": "14:50-15:10",
  "target_object": "手机",
  "expected_action": "search_recording_then_analyze"
}
```

### 4. 验证用例

- 正常查询：有录像、有目标、有结论。
- 无结果查询：该时间段没有目标或录像缺失。
- 模糊查询：用户只说“下午”或“门口”，需要追问。
- 权限异常：NVR 登录失败或通道不可访问。
- 低置信度：只能返回候选片段，不能武断判断身份。

## 能力上线节奏

1. 先做一个 NVR 品牌适配器，建议从客户最多的品牌开始。
2. 接通录像查询、片段拉取和通道映射。
3. 再训练专项能力：AI 找人、物品遗失、车辆检索。
4. 每个能力形成独立测试集。
5. 测试通过后打包成 AgentHub 能力版本。
6. 上传平台，标记版本号、适配范围和注意事项。

## 当前约定

- 摄像头实时接入默认 RTSP / ONVIF。
- 历史录像默认来自客户 NVR 录像主机。
- AI 找人属于录像回查之上的专项智能分析能力，需要单独展示和训练。
- 未完成品牌适配前，平台只收集查询条件，不承诺真实回放检索结果。
